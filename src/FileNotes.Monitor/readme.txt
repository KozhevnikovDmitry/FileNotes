FileNotes.Monitor - инструкция по установке, настройке и использованию.
1. Предварительные шаги.
	1.1. Приложение требует установленного .Net Framework 4.5.
	1.2. Для успешной сборки приложения необходимо добавить каталог с утилитой msbuild.exe в переменную среды PATH. Компьютер >> Свойства >> Дополнительные параметры системы >> Переменные среды >> Системные переменные >> Path. К строке необходимо добавить путь к каталогу ";C:\Windows\Microsoft.NET\Framework\v4.0.30319".
2. Сборка приложения.Создание структуры базы данных.
	2.1. Для сборки приложения необходимо запустить файл src/FileNotes.Monitor/build.cmd
	2.2. Готовый дистрибутив программы будет скомпонован в каталог src/FileNotes.Monitor/distr
	2.3. Комплектность дистрибутива:
		2.3.1. Scripts/db-script.sqlbuild.cmd
		2.3.2. Dapper.dll
		2.3.3. Dapper.pdb
		2.3.4. Dapper.xml
		2.3.5. FileNotes.Domain.dll
		2.3.6. FileNotes.Monitor.exe
		2.3.7. FileNotes.Monitor.exe.config
		2.3.8. readme.txt
3. Создание структуры базы данных.
	3.1. В каталоге дистрибутива находится файл с SQL-скриптом создания БД - Scripts/db-script.sql
	3.2. Для того, чтобы создать структуры базы данных необходимо выполнить данный скрипт на выбранном инстансе SQL Server.
	3.3. Скрипт создаст базу данных, таблицы, пользователя для приложения, а также выдаст необходимые права доступа.
	3.4. Ожидается, что на сервере организована аутентификация SQL Server Mode.
4. Настройка окружения.
	4.1. Приложение работает с двумя каталогами: каталог мониторинга и каталог обрабтанных файлов.
	4.2. Оба каталога необходимо создать или выбрать из уже существующих каталогов.
5. Настройка конфигурации.
	5.1. В каталоге дистрибутива находится xml-файл конфигурации приложения - FileNotes.Monitor.exe.config.
	5.2. Необходимо отредактировать данный файл для корректной работы программы.
	5.3. Настройка каталога мониторинга:
		5.3.1. Секция configuration >> секция appSettings >> элемент add по ключу(key) sourceFolder >> в значение атрибута value необходимо указать абсолютный или относительный(относительно файла FileNotes.Monitor.exe) путь к выбраному каталогу мониторинга.	
	5.4. Настройка каталога отработанных файлов:
		5.4.1. Секция configuration >> секция appSettings >> элемент add по ключу(key) destinationFolder >> в значение атрибута value необходимо указать абсолютный или относительный(относительно файла FileNotes.Monitor.exe) путь к выбраному каталогу отработанных файлов.
	5.5. Настройка строки соединения с базой данных:
		5.5.1. Секция configuration >> секция connectionStrings >> элемент add по ключу(key) filenotes >> в значении атрибута value("Data Source={SERVER_NAME};User ID=filenotes;Password=till25for@") необходимо заменить {SERVER_NAME} на имя сервера, на котором был исполнен скрипт в пункте 3.1.
6. Запуск приложения.
	6.1. Для запуска приложения необходимо запустить файл FileNotes.Monitor.exe в каталоге дистрибутива.
